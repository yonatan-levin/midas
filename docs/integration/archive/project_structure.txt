MIDAS PROJECT STRUCTURE
========================

Project: Midas - Financial Technology Platform
Language: Go (Golang)
Architecture: Clean Architecture with Dependency Injection

This document outlines the structure of the Midas project, a Go-based financial technology platform. It follows Clean Architecture principles to ensure a separation of concerns, making the system maintainable, scalable, and testable.

ARCHITECTURE SUMMARY
====================

This Go-based financial technology platform follows Clean Architecture principles:

1.  **Domain Layer** (internal/core/): Contains business entities and interfaces, independent of other layers.
2.  **Application Layer** (internal/services/): Implements use cases and business logic, orchestrating domain objects.
3.  **Infrastructure Layer** (internal/infra/): Implements interfaces defined in the core layer to interact with external systems like databases, caches, and external APIs.
4.  **API Layer** (internal/api/): Exposes the application's functionality via a RESTful API.
5.  **Dependency Injection** (internal/di/): Wires all components together, promoting loose coupling and testability.

KEY FEATURES:
- Financial data processing and cleaning with an extensible rule engine and AI integration.
- Core financial valuation models (DCF, WACC, Growth).
- Integration with external data sources (SEC EDGAR, Market Data APIs).
- Caching (Redis) and resilience patterns (Circuit Breaker, Retry).
- Comprehensive testing suite including unit, integration, performance, and fuzz tests.
- CI/CD pipelines for automated testing and deployment.
- Background job scheduling for data fetching.

The project is actively developed, with a clear separation of concerns that supports maintainability and future expansion.
