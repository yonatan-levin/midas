# Damodaran sector-multiple annual refresh (RM-2 Phase 2)

The `revenue_multiple` valuation model resolves its EV/Revenue multiple primarily
from Aswath Damodaran's NYU-Stern annual sector table ("Price and Value to
Sales", `psdata.xls`), keyed by SEC SIC code. This runbook covers the annual
refresh of the committed table.

## What is committed vs local

| Artifact | Tracked? | Role |
|---|---|---|
| `config/damodaran_sector_multiples.json` | yes | **Canonical** Damodaran industry → EV/Sales table + `dataset_date`. Regenerated by the tool. |
| `config/sic_to_damodaran.json` | yes | **Hand-maintained** SIC → Damodaran-industry crosswalk. NOT touched by the tool. |
| `data/damodaran/<date>/psdata.xls` | no (gitignored) | Local raw `.xls` snapshot for provenance. |

Both JSON configs are embedded via `config/embed.go` (`//go:embed`) — a file
omitted from that directive is unreadable in production/replay.

## When to refresh

Damodaran publishes updated datasets in early January each year. Refresh once the
new `psdata.xls` is live (the source `Date updated:` cell advances, e.g.
`2026.01`).

## Procedure

1. **Run the refresh tool** from the repo root with a contact-bearing User-Agent:

   ```bash
   go run ./cmd/refresh-damodaran -ua "midas-dcf/refresh-damodaran (you@example.com)"
   ```

   It fetches `psdata.xls`, snapshots the raw bytes to `data/damodaran/<date>/`,
   validates the workbook shape (sheet `Industry Averages`; header labels
   `Industry Name` / `EV/Sales`), and rewrites
   `config/damodaran_sector_multiples.json`. A layout change in the source file
   makes the shape guard exit non-zero rather than write garbage.

2. **Run the integrity gate** to catch industry-name renames that orphan
   crosswalk entries:

   ```bash
   go test ./internal/services/valuation/models/ -run Integrity
   ```

   If an industry the crosswalk references was renamed/removed in the new
   dataset, `TestSICToDamodaran_ReferentialIntegrity` fails. Update
   `config/sic_to_damodaran.json` (the hand-maintained file) to the new name.

3. **Review the diff.** `git diff config/damodaran_sector_multiples.json` should
   show updated `dataset_date` + EV/Sales value drift only. New/removed
   industries are expected year-over-year.

4. **Run the full suite** and commit the regenerated config (+ any crosswalk
   fixups):

   ```bash
   go test ./...
   ```

## Extending SIC coverage

Coverage is **partial by design**: unmapped SICs degrade gracefully to the
Phase 1 classifier bucket (`industry_multiples.json`), then the `default` 2.0×.
To map an additional SIC, add an entry to `config/sic_to_damodaran.json` whose
value EXACTLY matches a key in `damodaran_sector_multiples.json` (the integrity
gate enforces this). No code change is required.

## Deviations from the original plan (documented)

- **Dataset date is `YYYY.MM`, not an Excel serial.** The original plan assumed
  cell B1 held an Excel date serial decoding to `2026-01-05`. The real file
  carries a `"YYYY.MM"` string (`"2026.01"`) in the `Date updated:` cell — the
  day component is absent. The tool canonicalizes to the first of the month
  (`2026-01-01`); that is the value surfaced as `industry.multiple_source =
  "Damodaran 2026-01-01"`.
- **Integrity check substitutes for the 60-day request-log check.** The tracker
  asked for "every SIC seen in the last 60 days of request logs is covered". No
  request-log corpus exists in CI, so the static referential-integrity test
  (`TestSICToDamodaran_ReferentialIntegrity` + `TestDamodaranDatasetDate_Parses`)
  guarantees instead that the crosswalk can never reference a non-existent
  Damodaran industry — the load-bearing safety property.
- **Phase 1 buckets retained as a fallback tier** (not deleted as the tracker
  originally proposed) — a human decision for zero-regression: any path that does
  not resolve a Damodaran multiple is bit-for-bit identical to pre-Phase-2.

## Dependency note

`psdata.xls` is legacy OLE2/BIFF8 `.xls` (not `.xlsx`). `excelize` cannot read
it; the tool uses `github.com/extrame/xls`, confined to `cmd/refresh-damodaran`
and NOT imported by `cmd/server` (no `.xls` dependency on the request path).
