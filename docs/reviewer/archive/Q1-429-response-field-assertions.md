# Q1: Incomplete 429 Response Field Assertions

**Status:** OPEN
**Severity:** MINOR
**Found by:** QA (2026-04-14)
**Location:** `internal/api/server_test.go`, `TestServer_rateLimitMiddleware` "denied" case

## Description

The 429 rate-limit response test only asserts `error.code` and `rate_limit` presence. Per CONTRACTS.md, the 429 response also includes: `error.message`, `error.type`, `rate_limit.remaining`, `rate_limit.reset_time`, `rate_limit.retry_after`, `timestamp`, `path`, and `method`. These 8 fields are not individually asserted.

## Recommended Fix

Add assertions to the existing test case:

```go
assert.Equal(t, "Rate limit exceeded", errorObj["message"])
assert.Equal(t, "rate_limit_error", errorObj["type"])
assert.NotNil(t, body["timestamp"])
assert.NotNil(t, body["path"])
assert.NotNil(t, body["method"])
```

## Risk if Not Fixed

Low. The 429 response is generated by the rate limiter, not by `respondWithError`. Any structural change to the rate limit response would go unnoticed by tests.
